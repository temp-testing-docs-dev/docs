The OpenAPI specification provides a standardized description of our REST API. It outlines all available endpoints, request/response formats, authentication methods, and other important details for integrating with the Vapi platform.

## Key Benefits

- **Interactive Documentation:** Explore and test API endpoints directly through tools like Swagger UI.
- **Code Generation:** Automatically generate client libraries in various programming languages.
- **API Validation:** Ensure your requests match the expected format before sending them.

## Versioning

Our OpenAPI specification is versioned to maintain backwards compatibility. The current version is available at the URL above, while older versions can be accessed by appending the version number (e.g. `/api-json/v1`).

## Using the Specification

You can use the OpenAPI specification in several ways:

1. **Direct Integration:** Import the JSON/YAML file into your preferred API development tools.
2. **Swagger UI:** Explore the interactive documentation at [https://api.vapi.ai/api](https://api.vapi.ai/api).
3. **Code Generation:** Use tools like OpenAPI Generator to create client libraries for your preferred programming language.
