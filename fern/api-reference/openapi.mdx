## Using the OpenAPI Specification

### Interactive API Explorer

You can explore and test our APIs directly using our interactive documentation:

- **Swagger UI**: [https://api.vapi.ai/api](https://api.vapi.ai/api)
- **Raw OpenAPI JSON**: [https://api.vapi.ai/api-json](https://api.vapi.ai/api-json)

### SDK Generation

The OpenAPI specification can be used to generate SDKs in multiple programming languages:

```bash
# Generate TypeScript SDK
openapi-generator generate \
  -i https://api.vapi.ai/api-json \
  -g typescript-axios \
  -o ./vapi-typescript-sdk

# Generate Python SDK
openapi-generator generate \
  -i https://api.vapi.ai/api-json \
  -g python \
  -o ./vapi-python-sdk

# Generate Java SDK
openapi-generator generate \
  -i https://api.vapi.ai/api-json \
  -g java \
  -o ./vapi-java-sdk
```

## Validation and Testing

Use the OpenAPI specification for API validation and automated testing:

### Schema Validation

```javascript
import Ajv from 'ajv';
import fetch from 'node-fetch';

// Load OpenAPI schema
const schemaResponse = await fetch('https://api.vapi.ai/api-json');
const openApiSchema = await schemaResponse.json();

// Validate API responses
const ajv = new Ajv();
const validateAssistant = ajv.compile(
  openApiSchema.components.schemas.Assistant
);

const isValid = validateAssistant(assistantData);
if (!isValid) {
  console.error('Validation errors:', validateAssistant.errors);
}
```

### API Testing

```typescript
import { OpenAPIV3 } from 'openapi-types';

// Load and parse OpenAPI specification
const spec: OpenAPIV3.Document = await loadOpenAPISpec();

// Generate test cases from schema
function generateTestsFromSchema(spec: OpenAPIV3.Document) {
  Object.entries(spec.paths).forEach(([path, pathItem]) => {
    Object.entries(pathItem).forEach(([method, operation]) => {
      if (operation.operationId) {
        it(`should handle ${method.toUpperCase()} ${path}`, async () => {
          // Generate test based on schema
          const testData = generateTestData(operation);
          const response = await apiCall(method, path, testData);
          expect(response).toMatchSchema(operation.responses['200']);
        });
      }
    });
  });
}
```

## Version Management

The OpenAPI specification is versioned alongside our API releases:

- **Current Version**: Available at `/api-json`
- **Version-Specific**: Future versioned endpoints will be available at `/api-json/v{version}`
- **Changelog**: API changes are documented in our [release notes](/changelog)
